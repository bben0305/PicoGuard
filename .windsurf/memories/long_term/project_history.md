## 🔧 关键技术决策

### 通信协议选择
- **决策**: HTTP POST + JSON
- **原因**: 简单、可靠、易于调试
- **替代方案**: MQTT (过于复杂)

### 数据库选择
- **决策**: SQLite + SQLAlchemy
- **原因**: 轻量级、无需额外配置
- **替代方案**: PostgreSQL (过度设计)

### 部署平台
- **决策**: Railway
- **原因**: 简单、免费额度充足
- **替代方案**: AWS/Heroku (成本考虑)

## 📚 经验教训

### 成功经验
1. **渐进式开发**: 先让基础功能工作，再优化
2. **详细日志**: 关键时刻的调试日志至关重要
3. **分层测试**: 硬件 → 通信 → 后端 → 前端逐层验证

### 失败教训
1. **缓冲区大小**: 默认 UART 缓冲区不足以处理完整 HTTP 请求
2. **时序问题**: ESP01 需要足够的冷静期处理数据
3. **透传模式误解**: 非透传模式不需要 +++ 退出
