# PicoGuard 记忆系统架构

## 🧠 记忆分区设计

### 📂 文件结构
```
.windsurf/memories/
├── README.md              # 本文件 - 记忆系统说明
├── tech_stack.json        # 技术栈配置和关键修复
├── issue_log.md           # 问题记录和解决方案
├── short_term/            # 短期记忆区
│   └── current_session.md # 当前会话状态
├── long_term/             # 长期记忆区
│   └── project_history.md # 项目历史和技术决策
└── work_memory/           # 工作记忆区
    └── current_tasks.md   # 当前任务进展
```

## 🔄 记忆检索机制

### 检索优先级
1. **工作记忆** (`work_memory/`) - 当前任务上下文
2. **短期记忆** (`short_term/`) - 会话状态和临时信息
3. **长期记忆** (`long_term/`) - 项目历史和技术决策
4. **技术栈** (`tech_stack.json`) - 技术细节和配置

### 自动更新规则
- **每次重大修复后**: 更新所有相关记忆文件
- **会话结束时**: 清理短期记忆
- **任务完成时**: 更新工作记忆状态
- **技术变更时**: 更新技术栈配置

## 📋 记忆内容分类

### 🔧 技术记忆 (`tech_stack.json`)
- 技术栈配置
- 关键修复方案
- 部署信息
- 硬件规格

### 🐛 问题记忆 (`issue_log.md`)
- 问题描述和症状
- 根因分析
- 解决方案
- 关键代码片段
- 状态跟踪

### 📅 短期记忆 (`short_term/`)
- 当前会话状态
- 临时调试信息
- 会话进展
- 待处理事项

### 📚 长期记忆 (`long_term/`)
- 项目历史
- 技术决策
- 架构演进
- 经验教训

### 🎯 工作记忆 (`work_memory/`)
- 当前任务列表
- 任务进展状态
- 阻塞问题
- 下一步行动

## 🔄 记忆同步流程

### 每次重大修复后
1. **更新 issue_log.md** - 记录新问题和解决方案
2. **更新 tech_stack.json** - 更新技术细节
3. **更新工作记忆** - 标记任务完成
4. **更新短期记忆** - 记录会话进展
5. **必要时更新长期记忆** - 重要技术决策

### 自动检查清单
- [ ] 问题是否完全解决？
- [ ] 解决方案是否记录在 issue_log.md？
- [ ] 技术细节是否更新在 tech_stack.json？
- [ ] 上下文文件是否同步？
- [ ] 是否有遗漏的调试代码需要清理？

## 🎯 使用指南

### 对于 AI 助手
1. **每次会话开始**: 读取工作记忆了解当前任务
2. **遇到技术问题**: 查阅 issue_log.md 和 tech_stack.json
3. **需要项目背景**: 查阅长期记忆
4. **会话结束**: 更新所有相关记忆文件

### 对于开发者
1. **查看项目状态**: 阅读 README.md 和工作记忆
2. **了解技术细节**: 查阅 tech_stack.json
3. **排查问题**: 查阅 issue_log.md
4. **了解项目历史**: 查阅长期记忆

## 🚀 最佳实践

### 记忆更新原则
- **及时性**: 问题解决后立即更新
- **完整性**: 包含问题、根因、解决方案、关键代码
- **一致性**: 所有记忆文件保持同步
- **可读性**: 使用清晰的格式和注释

### 记忆检索原则
- **相关性**: 优先检索最相关的记忆
- **时效性**: 优先检索最新的信息
- **完整性**: 确保不遗漏重要上下文
- **效率**: 快速定位所需信息
